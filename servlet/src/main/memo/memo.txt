* httpServlet
1. request
    - 서버 tcp/ip 대기, 소켓 연결
    - Http 요청 메시지를 파싱해서 읽기
    - POST 방식, /save(해당 url) 인지
    - Content-Type 확인
    - Http 메시지 바디 내용 피싱
        -username, age 등의 데이터를 사용할 수 있게 파싱
    - 저장 프로세스 실행
2. response
    - HTTP 응답 메시지 생성 시작
        - http 시작 라인 생성
        - Header 생성
        - 메시지 바디에 HTMl 생성에서 입력
    - TCP/IP에 응답 전달, 소켓 종료
3. 비즈니스 로직 실행
    - 데이터베이스에 저장 요청
